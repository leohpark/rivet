## Overview

Text Nodes are string constants used to pass data to other nodes. The text can be written and stored within the Editor, or passed in using dynamically created Inputs through the use of double-handlebar notation within the Editor.

## Inputs are Coerced to Strings

If the Text Node receives input from another Node that does not produce strings from it's output port, the [coerceType](packages/core/src/utils/coerceType.ts) is used to convert the input into a string-type object. For example, a Prompt Node outputs a "chat-message" object, which is a dictionary containing a "type" and "message" key/value pairs. coerceType will convert a "chat-message" object into just the "message" value as a string. 

If you would like to preserves more of the original input data type, then you should probably use a [Prompt Node](packages/docs/docs/node-reference/prompt.mdx), [Assemble Prompt Node](packages/docs/docs/node-reference/assemble-prompt.mdx), or [Array Node](packages/docs/docs/node-reference/array.mdx) instead of a text node. Or, use a [Prompt Node](packages/docs/docs/node-reference/prompt.mdx) after the text node to re-format the string into a chat-message object.
